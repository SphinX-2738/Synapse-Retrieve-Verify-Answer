SYNAPSE
Retrieve. Verify. Answer. — Gen AI Portfolio Project

What it does

Upload your PDFs (papers, manuals, reports), ask any question, and Synapse decides whether to retrieve from your documents or answer directly, generates a grounded answer with citations, then evaluates its own output for hallucination — re-running itself if it falls short.

The Agentic Loop

Five steps run on every question:

01
Plan — LLM decides at temperature 0.0 whether to search documents or answer directly
02
Retrieve — ChromaDB cosine search returns top 5 chunks; query is rephrased and retried if relevance < 0.6
03
Generate — answer is constrained to retrieved chunks only, with mandatory citations
04
Evaluate — a second LLM call judges groundedness, relevance, and citation accuracy. Hallucination caps the score at 5.0
05
Self-Correct — if overall score < 7, the agent retries up to 2 times before returning

Tech Stack

Python 3.11 FastAPI Groq LLaMA 3.3 70B Gemini Embeddings ChromaDB pdfplumber NLTK Render

Key Features

▸Agentic routing — knows when not to retrieve
▸Built-in hallucination detection with self-retry loop
▸Sliding-window conversation memory with auto-summarisation
▸Session-per-user ChromaDB isolation — no cross-user leakage
▸SSE streaming responses with full evaluation envelope on completion

Why I Built This

Most RAG demos retrieve blindly and hope the LLM stays grounded. Synapse decides, cites, and judges itself — because shipping agents that can defend their own answers is what separates a portfolio project from a production system.

How to use

01 Drag & drop one or more .pdf files to build your knowledge base — or skip this and ask a general question
02 Type your question in the input box, or hit a sample to see the agent in action
03 Press ASK SYNAPSE or Ctrl + Enter to run the agentic loop
04 Review the answer, citations, and self-evaluation scores below — every claim is traceable to a source page
